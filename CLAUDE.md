# CLAUDE.md

Guidance for Claude Code when working in this repository.

## Stack

- Astro 5 (static output, no SSR)
- Tailwind CSS v4 via `@tailwindcss/vite`
- MDX for blog posts (`@astrojs/mdx`)
- TypeScript
- Vanilla `<script>` blocks for interactivity (no React)
- Deployed on Cloudflare Pages (auto-deploy from `main`)

## Commands

```bash
npm install           # install deps
npm run dev           # dev server on :4321 (regenerates sprite first)
npm run build         # static output in dist/
npm run preview       # serve dist/ locally
npm run build:icons   # rebuild public/sprite.svg from icons/
npm run sync-books    # populate src/data/books.json + public/covers/ from Calibre
```

## Bookshelf data flow

The `/books` page reads `src/data/books.json`. Covers are committed JPEGs under
`public/covers/{id}.jpg`. Both are generated by `scripts/sync-books.mjs`, which
must run somewhere with access to the Calibre library (i.e., on the Hetzner
host — the filesystem is not available locally or in Cloudflare Pages build).

Flags:
- `--dry-run`  write files locally, do not touch git
- `--no-push`  commit but do not push

Env:
- `CALIBRE_LIBRARY`  path to Calibre library root (default `/calibre`)

The script orphan-cleans covers whose book IDs are no longer 5-star, compares
new JSON against committed JSON before committing, and pushes with a bot
identity.

On the server (`cloud-hil-1`), the script runs daily via
`personal-site-sync-books.timer` (see `hetzner/SERVER.md`). The repo is cloned
at `/opt/personal-site`; push auth is a per-repo SSH deploy key at
`/root/.ssh/personal_site_deploy` (write access) used via the
`github-personal-site` Host alias in `/root/.ssh/config`.

### Adding a new dynamic content source

To add another external content source (e.g., photos from Immich, podcasts):

1. Add `scripts/sync-X.mjs` modeled on `sync-books.mjs`: fetch from source →
   write `src/data/X.json` (+ assets in `public/X/` if applicable).
2. Diff new JSON against the committed copy before writing; orphan-clean
   assets whose IDs are no longer in the new set.
3. Commit as a bot identity; push with rebase-on-conflict retry.
4. Wire it to a `personal-site-sync-X.{service,timer}` pair on the host where
   the data source lives (typically `cloud-hil-1`). See the books timer for
   the unit file shape.

## Conventions

- **Styling**: Tailwind v4 with custom `@utility` rules (`text-h1`, `text-h2`,
  etc.) defined in `src/styles/app.css`. Dark mode via `prefers-color-scheme`
  CSS variables — no JS toggle.
- **Icons**: add an SVG to `icons/`, rebuild the sprite with
  `npm run build:icons`, consume via `<Icon name="your-icon" />` which emits
  `<svg><use href="/sprite.svg#id" /></svg>`.
- **Interactivity**: prefer vanilla `<script>` blocks inside Astro components
  over React islands. Examples: `MobileNav.astro`, `Scrollspy.astro`.
- **SEO**: every page goes through `BaseLayout.astro`, which enforces `title`
  and `description` props at the type level. Canonical, OG, Twitter, and
  sitemap are handled there.
- **Blog**: drafts (`draft: true` in frontmatter) are visible in dev, hidden
  in prod. RSS at `/rss.xml`. Syntax highlighting via Shiki dual-theme.
- **Fonts**: self-hosted via `@fontsource` packages, imported in
  `BaseLayout.astro`.

## Deployment

Cloudflare Pages builds on push to `main`:
- Build command: `npm run build`
- Output dir: `dist`
- Node version: pinned via `.node-version`

Cache-Control overrides live in `public/_headers`.
